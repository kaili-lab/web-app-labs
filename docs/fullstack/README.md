# 现代 Web / 全栈主线目录

这里承载现代应用 / 全栈认知主线。

## 这条主线要回答什么
这个目录重点回答的是：
- 一个现代 Web 应用从简单到复杂，会遇到哪些典型问题
- 这些问题如何从产品目标、页面交互、前后端协作、数据流动、运行部署一路串起来
- 常见方案分别解决什么问题、什么时候值得引入、代价是什么

## 这条主线不是什么
- 不是技术栈名词墙
- 不是框架功能手册
- 不是“把 React、Next.js、Node.js 各写一份百科”

`React`、`Next.js`、`Node.js` 等在这里默认是实现手段，不是一级主线本身。

## 当前整理原则
这个目录优先按问题域组织，而不是按技术名词组织。

例如后续更适合逐步回答这类问题：
- 页面和交互为什么会复杂起来
- 渲染和数据获取为什么会成为问题
- 状态共享为什么会出现
- 前后端边界为什么会变化
- 认证、性能、可观测性、部署为什么会逐步重要

## 当前状态说明
`docs/fullstack/` 下已经存在一些按技术栈拆分的材料，例如 `react/`、`nextjs/`。

当前把它们视为过渡性笔记或参考材料，不把它们当成这个目录最终的一级结构方向。

## 和其他文档的关系
- 如果你想看仓库级目标和边界 -> 回到 `项目记忆.md`
- 如果你想看通用整理方法 -> 回到 `领域知识整理方法论.md`
- 如果你想看 AI 作为重点专题线如何挂接 -> 看 `docs/ai-application/README.md`

## 当前建议的使用方式
1. 先读 `项目记忆.md`
2. 再读 `领域知识整理方法论.md`
3. 再用这里的文档逐步搭出现代应用主线骨架
4. 需要看具体技术材料时，再进入 `react/`、`nextjs/` 等过渡性笔记
